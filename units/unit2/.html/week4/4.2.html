Lesson

What is Binary Encoding?

Computers do not understand human languages or numbers the way we do. Instead, they store and process all types of data—numbers, text, images, audio, and video—using a simple system of 0s and 1s known as binary encoding. 

Why Do Computers Use Binary? 

Computers are made up of millions of tiny transistors, which are electronic switches that turn ON and OFF. This ON/OFF system matches perfectly with binary encoding, where: 

  • 1 represents ON (electricity flowing)
  • 0 represents OFF (no electricity flowing) 

By combining many 0s and 1s, computers can store complex data such as images, music, and videos. 

Real-Life Example: Light Bulbs 

Imagine a row of light bulbs where each bulb can be either ON or OFF. If you use a sequence of ON/OFF states, you can represent numbers or even words. 

For example, if three bulbs are ON (1) and one is OFF (0), this could mean a certain number in binary. This is how computers process and store data at a fundamental level. 

The mechanisms by which data are stored in binary form

(a) Storing Numbers in Binary 

Numbers are stored in binary using the base-2 number system, where each digit represents a power of 2. 

Computers perform all calculations using binary arithmetic, adding, subtracting, and multiplying binary numbers. 

Impact on Storage and Retrieval: 

  • More bits allow larger numbers to be stored. For example, an 8-bit number can store values from 0 to 255.
  • Numbers must be converted back to decimal for human understanding. 

Example: Converting 13 to Binary

13 in decimal = 1101₂

(b) Storing Text in Binary (Character Encoding): Computers store letters, numbers, and symbols as numbers, which are then stored in binary. Character encoding schemes like ASCII and Unicode define how text is represented. 

Each letter you type on a keyboard is stored as a binary code, retrieved when needed, and displayed on the screen. 

Impact on Storage and Retrieval: 

  • Small text files require minimal storage.
  • Unicode supports multiple languages, making it essential for international applications. 

Example: Storing ‘A’ in Binary 

  • ASCII assigns ‘A’ the decimal value 65.
  • 65 in binary = 01000001₂ 

(c) Storing Images in Binary: Images are stored as pixels, with each pixel represented by a binary code that defines its color and brightness. 

For color images, each pixel is stored as a combination of Red, Green, and Blue (RGB) values, where - 8 bits per color → 24 bits per pixel → 16 million colors possible! 

Impact on Storage and Retrieval: 

  • Higher resolution images require more binary data. 
  • Compression (JPEG, PNG) reduces storage size for faster retrieval. 

Example: Storing a Black-and-White Image 

A 1-bit image uses 0 for black and 1 for white. 

Example Grid

This simple grid could represent a small black-and-white image stored in binary.

(d) Storing Audio in Binary: Sound waves are continuous (analog) but computers store them in discrete steps using sampling. 

Impact on Storage and Retrieval: 

  • Higher sample rates improve sound quality but take up more space.
  • Compression (MP3) reduces file sizes for easier storage and transmission. 

Example: How a Song is Stored 

  1. A microphone captures sound waves.
  2. The wave is measured thousands of times per second (sampling rate).
  3. Each measurement is stored as a binary value. 

CD-quality audio = 44,100 samples per second, each stored as 16-bit binary numbers.

(e) Storing Video in Binary: Videos are stored in binary as a sequence of images (frames) combined with an audio track. Each frame is a still image, and when displayed rapidly—typically 30 or 60 frames per second—they create the illusion of motion. The audio track is stored separately and synchronised with the video frames.

Impact on Storage and Retrieval:

  • Each video frame is a binary-encoded image; more frames per second create smoother motion.
  • Raw video files are very large; compression is essential for practical storage and transmission.
  • Common compression formats (e.g., MP4, H.264) reduce file size by eliminating redundant data.
  • Streaming platforms (e.g., YouTube, Netflix) use adaptive streaming to adjust video quality based on internet speed, balancing performance and bandwidth use.

Example: Understanding Video as a Flipbook

Imagine a flipbook where each page shows a slightly different image. Flipping through the pages quickly makes the images appear to move. Similarly, in digital video, each frame is encoded in binary and played in sequence. A one-hour HD video (1920 × 1080 resolution) stored without compression could occupy hundreds of gigabytes.

To manage storage, video and audio data are compressed using formats such as MP4 and H.264. These remove redundant data while maintaining quality, significantly reducing file size.

Points to Remember: 
  • Videos are stored as a sequence of binary-encoded frames with an audio track.
  • More frames per second (fps) create smoother motion in videos.
  • Uncompressed videos take up large amounts of storage, making compression necessary.
  • MP4 and H.264 are widely used compression formats for efficient storage.
  • Streaming services adjust video quality dynamically based on internet speed. 

The Impact of Binary Encoding on Data Storage and Retrieval

(a) Storage Efficiency
  • More bits require more storage, making efficiency a priority.
  • Compression reduces file size, improving storage efficiency.
  • Lossless compression keeps all data intact, while lossy compression removes unnecessary details.
  • Cloud storage services use compression to optimize space and reduce transfer times. 

Storage efficiency explained in detail

The efficiency of binary encoding directly influences how much storage space is needed to store digital data. Every type of digital file—whether text, images, audio, or video—requires a specific number of bits to represent the information. The more bits used, the larger the file size. Since excessive storage requirements can be costly and impractical, various techniques have been developed to optimize how binary data is stored. 

One of the most important techniques for improving storage efficiency is compression. Compression reduces the number of bits required to represent data while preserving its essential characteristics. There are two main types of compression: lossless compression and lossy compression. Lossless compression, used in formats like ZIP and PNG, reduces file size without losing any data. This is useful for text documents and images where accuracy is critical. Lossy compression, used in formats like MP3, JPEG, and MP4, achieves much higher compression rates by discarding unnecessary details that the human eye or ear may not notice. For example, a JPEG image file is significantly smaller than an uncompressed RAW image file, yet it still appears visually similar to most viewers. 

A practical example of how storage efficiency affects real-world applications can be seen in cloud storage services like Google Drive and Dropbox. These services employ advanced compression algorithms to store large amounts of data efficiently, allowing users to save and retrieve their files quickly while minimizing storage costs. Without compression, cloud storage would be significantly more expensive, and transferring files over the internet would be much slower. 
(b) Data Retrieval Speed 
  • Retrieval speed depends on storage size and medium (HDD vs. SSD).
  • SSDs retrieve data faster because they use flash memory.
  • Indexed databases improve search and retrieval efficiency.
  • Caching in video streaming helps prevent buffering and speeds up playback. 

Data retrieval speed explained in detail

The time it takes to retrieve stored data depends on several factors, including file size, storage medium, and the efficiency of the retrieval process. Larger files take longer to access and open, especially if stored on traditional hard disk drives (HDDs), which rely on moving mechanical parts to read and write data. In contrast, solid-state drives (SSDs), which store data electronically in flash memory, can access binary data almost instantly, making them significantly faster. 

The structure of stored data also impacts retrieval speed. In databases, for example, information is often stored in an indexed format, which allows for faster searches and retrieval. Consider the difference between finding a book in a traditional library versus using a search engine in a digital library. In a physical library, one must manually browse bookshelves, which is time-consuming. However, in a digital system, search functions retrieve information instantly by looking up pre-indexed binary data. This principle is used in search engines like Google, which scans massive databases of web pages and returns relevant results in milliseconds. 

In video streaming services, retrieval speed is crucial to prevent buffering. When a user selects a video on Netflix or YouTube, the platform retrieves and loads the video data in chunks, ensuring a seamless viewing experience. These services use caching techniques, where frequently accessed video data is stored temporarily in faster-access memory, reducing the need to retrieve the full video file repeatedly from storage. Without such optimizations, streaming high-quality videos in real time would be extremely slow and inefficient. 
(c) Data Security 
  • Encryption secures data by converting it into a binary-coded secret.
  • End-to-end encryption protects private communications.
  • Error detection techniques like parity bits ensure data accuracy.
  • Secure data storage is essential for banking, messaging, and healthcare.

Data security explained in detail

Binary encoding also plays a critical role in data security, ensuring that stored and transmitted information remains protected from unauthorized access. One of the most important security measures is encryption, a process that converts readable data into a coded format that can only be accessed by authorized users. Encryption algorithms such as AES (Advanced Encryption Standard) use complex binary transformations to encode data, making it virtually impossible for hackers to decipher without the correct decryption key. 

For example, when you log in to an online banking system, your password and transaction details are encrypted in binary before being sent over the internet. This ensures that even if a hacker intercepts the data, they will only see a meaningless sequence of binary digits rather than readable text. Encryption is also widely used in secure messaging apps like WhatsApp and Signal, where end-to-end encryption protects private conversations from being accessed by third parties. 

Another key aspect of binary-based security is error detection and correction, which ensures that data remains accurate during storage and transmission. Techniques such as parity bits and checksums detect errors in data packets sent over networks, allowing corrupted data to be identified and corrected before reaching the recipient. This is particularly important for applications like financial transactions, airline reservation systems, and medical databases, where data integrity is essential. 

Summary

Key Takeaways 

  • Binary encoding is used for all types of digital data.
  • Numbers, text, images, audio, and video are stored as binary patterns. 
  • Storage efficiency and retrieval speed depend on file formats, compression, and hardware. 
  • Real-world applications include file storage, encryption, and video streaming.
