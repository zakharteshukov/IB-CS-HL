Introduction

Learning Objectives:

  • Explain how operating systems abstract hardware complexities to efficiently manage system resources such as CPU, memory, storage, and input/output devices.

An operating system (OS) is the software that acts as an intermediary between the user and the hardware, abstracting the complexities of system resources such as the CPU, memory, storage, and input/output (I/O) devices. Without an operating system, users would have to manually manage memory allocation, CPU scheduling, and device communication, which would make computers impractical for daily use.

Operating systems ensure efficient resource allocation, allowing multiple applications to run simultaneously while optimizing system performance and ensuring stability. Through features like process scheduling, memory management, file system organization, and device drivers, the OS provides an interface for users and applications to interact with the hardware seamlessly.

Lesson

CPU Management and Process Scheduling

The CPU is the brain of the computer, executing instructions from various programs. The OS abstracts CPU complexity by:

  • Scheduling multiple processes efficiently using algorithms such as Round Robin, First-Come-First-Served, and Priority Scheduling.
  • Time-sharing by dividing CPU time among processes to ensure smooth multitasking.
  • Handling process execution by switching between tasks using context switching, which allows seamless user experience without manual intervention.

Memory Management

In operating systems, memory manager is the process of managing available space to allow different applications to be stored..

It determines how memory is allocated among competing processes, deciding which gets memory, when they receive it, and how much they are allowed.

Types of memory management as used by OS

Segmentation: Memory segmentation is a computer (primary) memory management technique of division of a computer's primary memory into segments or sections. In a computer system using segmentation, a reference to a memory location includes a value thament is the function responsible for managing the computer's primary memory identifies a segment and an offset (memory location) within that segment.

Segmented Memory

  • Segments are swapped between disc and main memory as needed
  • Program segments correspond to blocks of program code such as procedures or functions
  • Data segments correspond to data structures such as stacks, queues or graphs
  • Segments vary in size
  • The operating system knows start and size of each segment in physical memory
  • Each segment is atomic, either the whole segment is in RAM, or none of the segment is in RAM
  • A segment in memory can only be replaced by a segment of the same size or smaller
  • Segmentation can result in memory fragmentation; a lot of small segments with gaps in between
  • Large segments may not be allowed into memory very often
  • Segments can be “pushed together” to limit fragmentation and allow large segments to be loaded

Paging: paging is a memory management scheme by which a computer stores and retrieves data from secondary storage for use in the main memory. In this scheme, the operating system retrieves data from secondary storage in same-size blocks called pages.

View:

Paged Memory

  • Memory is split up into small, equal sized sections called pages (or page frames)
  • A single application may occupy multiple pages, which are not necessarily contiguous
  • Each application program has its own view of the memory, known as logical memory
  • A page table records where the different pages of a program are located in physical memory
  • Unused pages may be paged out to a swap file on disc to make room for others
  • Pages are paged in when needed again
  • Supplementing physical memory with secondary storage is known as virtual memory
  • When memory is low, excessive swapping can lead to disc thrashing and degrade performance

Swapping: Swapping is a mechanism in which a process can be swapped temporarily out of main memory (or move) to secondary storage (disk) and make that memory available to other processes. At some later time, the system swaps back the process from the secondary storage to main memory.

Operating System Scheduling

Time Slicing: Time slicing Links to an external site. is a scheduling mechanism/way used in time sharing systems. It is also termed as Round Robin scheduling. The aim of Round Robin scheduling or time slicing scheduling is to give all processes an equal opportunity to use CPU. In this type of scheduling, CPU time is divided into slices that are to be allocated to ready processes. Short processes may be executed within a single time quantum. Long processes may require several quanta.

Priorities: Priority scheduling is a non-preemptive algorithm and one of the most common scheduling algorithms in batch systems.

Each process is assigned a priority. Process with the highest priority is to be executed first and so on.

Processes with the same priority are executed on a first come first served basis.

Priority can be decided based on memory requirements, time requirements or any other resource requirement.

Real-World Example: Multitasking on a Smartphone

A smartphone allows you to stream music, browse the web, and receive notifications simultaneously. The OS manages CPU time, ensuring that no single app consumes excessive resources, providing a smooth user experience.

Memory Management and Address Space Allocation

Memory management ensures that different applications can run without interfering with each other. The OS manages:

  • Allocation and deallocation of memory when programs start and stop.
  • Virtual memory, which allows the system to use storage as an extension of RAM when physical memory is full.
  • Segmentation and paging, which divide memory into efficient blocks to prevent fragmentation.

Real-World Example: Running Multiple Applications on a Computer

When opening multiple browser tabs, a word processor, and a media player, the OS dynamically allocates RAM to each application, ensuring efficient performance while preventing crashes due to memory conflicts.

Storage Management and File Systems

The OS abstracts physical storage complexities by:

  • Organizing data into file systems such as NTFS, FAT32, ext4 to enable efficient retrieval.
  • Managing disk access through caching and prefetching techniques to improve read/write speeds.
  • Ensuring data security through encryption, permissions, and access control mechanisms.

Real-World Example: Cloud Storage and File Management

In cloud storage services like Google Drive or Dropbox, the OS abstracts the underlying file system by allowing users to drag and drop files without worrying about how data is stored, retrieved, or distributed across multiple servers.

Input and Output (I/O) Device Management

Operating systems manage various input/output devices like keyboards, mice, printers, displays, and network interfaces by:

  • Providing device drivers that enable software to communicate with hardware.
  • Handling multiple device requests through an I/O scheduling mechanism to ensure fairness and efficiency.
  • Implementing buffer management to handle data transfer speeds between fast and slow devices.

Real-World Example: Plug-and-Play USB Devices

When you connect a USB drive, the OS automatically detects and installs the necessary driver, allowing seamless access without manual configuration. The OS abstracts low-level hardware interactions, making USB usage effortless for users.

System Resource Allocation and Load Balancing

Efficient resource allocation ensures that no single application consumes all system resources, preventing performance bottlenecks. The OS:

  • Monitors system load and dynamically adjusts allocations to maintain performance.
  • Distributes processing tasks in multi-core processors to optimize workload efficiency.
  • Manages energy consumption in mobile devices by adjusting CPU power and screen brightness dynamically.

Real-World Example: Cloud Computing and Virtualization

Cloud platforms like AWS and Google Cloud run multiple virtual machines on the same physical hardware. The OS ensures efficient resource allocation, balancing CPU, memory, and network bandwidth for different cloud users.

Summary

Key Points to Remember

  • The OS abstracts hardware complexities, making computers user-friendly and efficient.
  • CPU management ensures fair and efficient multitasking through process scheduling.
  • Memory management allows multiple applications to run without interference.
  • Storage management organizes data, improves speed, and secures files.
  • I/O device management enables seamless communication between hardware and software.
  • Efficient resource allocation prevents system overload and optimizes performance.
