Introduction to the Fetch-Decode-Execute Cycle 

The fetch-decode-execute cycle is the fundamental process by which a CPU carries out instructions in a program. This cycle consists of three main steps: 

  1. Fetch – The CPU retrieves an instruction from memory by sending the address stored in the Program Counter (PC) to the memory unit via the Address Bus. The instruction is then transferred to the Instruction Register (IR) through the Data Bus.
  2. Decode – The instruction in the IR is analyzed by the Control Unit (CU), which interprets its meaning by extracting the opcode and operand. The CU determines the necessary control signals needed for execution.
  3. Execute – The CPU carries out the instruction by performing the required operation. If arithmetic or logical operations are needed, the Arithmetic Logic Unit (ALU) processes them. The results are then stored in the appropriate register or memory location as required. 

This cycle repeats continuously while a program is running, ensuring smooth and sequential execution of instructions.

Step-by-Step Breakdown of the Cycle

 

  1. Fetch Stage 
    • The Program Counter (PC) holds the address of the next instruction. 
    • The Address Bus carries this address to RAM (Memory). 
    • The Instruction Register (IR) stores the retrieved instruction. 
    • The PC is incremented to point to the next instruction. 
  1. Decode Stage
    • The Control Unit (CU) reads the instruction stored in the IR. 
    • The instruction is broken down into its opcode (operation) and operands (data or memory location). 
    • The CU determines the necessary actions and signals required to execute the instruction.
  2. Execute Stage
    • The Arithmetic Logic Unit (ALU) or other CPU components perform the required operation. 
    • Data may be fetched from memory if needed. 
    • The result may be stored in a register or written back to memory. 

Interaction Between Memory and Registers via Three Buses 

The communication between the CPU and memory involves three main buses, each playing a crucial role in transferring data and control signals between memory and CPU registers: 

  1. Address Bus – The CPU uses the Program Counter (PC) to hold the memory address of the instruction or data to be accessed. This address is placed on the address bus and sent to RAM or an I/O device. The Memory Address Register (MAR) temporarily holds this address before data retrieval. 
  2. Data Bus – Once the memory location is identified, the Memory Data Register (MDR) fetches the required instruction or data and transfers it to the Instruction Register (IR) or a general-purpose register in the CPU. If a result from an operation needs to be written back, the data bus carries it from the Accumulator (ACC) or another register to the specified memory location. 
  3. Control Bus – The Control Unit (CU) sends control signals, such as read or write commands, to coordinate data transfers. For example, a 'Memory Read' signal ensures that data from a specified memory address is loaded into the MDR, while a 'Memory Write' signal ensures that processed data is stored back in memory. 

These buses work together to facilitate efficient instruction execution, ensuring seamless interaction between memory and the CPU registers for high-speed processing.

Summary

Summary

  • The fetch-decode-execute cycle allows a CPU to process instructions sequentially. 
  • The CPU interacts with memory and registers using the address, data, and control buses. 
  • The cycle is continuous and essential for program execution.
